Алгоритм решения был выбран следующий:

шаг1) На основе списка телепередач (Program) составляем список событий (ProgramEvent), где одно событие – это момент времени включения/выключения какой либо программы.

шаг2) Сортируем список событий в хронологическом порядке.

шаг3) Создаем стэк программ с приоритетом. Эта структура будет отражать текущий набор телепрограмм для определенного момента времени, упорядоченная по приоритету телепрограмм. Используя стэк программ и список событий, мы имеем возможность промоделировать все изменения состояний стэка программ для каждого события.

шаг4) На основе стэка программ и списка событий моделируем изменения состояния телеэфира для каждого события, и, когда верхушка стэка программ меняется (верхушка – позиция для самой приоритетной программы), то записываем действие по переключению канала на новую приоритетную программу в список действий(Action).

шаг5) Из-за особенностей работы алгоритма с (шага1) до (шага5) в списке действий могут появиться действия, совершенные в один момент, и соседствующие действия, включающие один и тот же канал. Отфильтровываем их.

Готово. На выходе получаем список действий по переключению каналов в определенные моменты времени.
